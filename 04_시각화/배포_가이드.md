# Vercel 배포 가이드

> Vercel CLI는 Node 18+ 필요해서 현재 환경(Node 16)에서 동작 안 함.
> 대신 **Vercel 웹 UI + GitHub 연동** 경로로 진행. 빌드는 Vercel 클라우드(Node 20)에서 돌아감.

---

## 1️⃣ Vercel 웹에서 프로젝트 import

1. https://vercel.com/new 접속 (GitHub 계정으로 로그인 — 무료)
2. **Import Git Repository** → `andypsh/datacenter_select` 옆 **Import** 클릭
3. 첫 진입 시 GitHub 앱 설치 권한 요청 → Authorize

## 2️⃣ Configure Project 설정 (★중요★)

| 항목 | 값 |
|---|---|
| Project Name | `datacenter-contest-v2` (혹은 원하는 이름) |
| Framework Preset | **Vite** (자동 감지됨) |
| **Root Directory** | `04_시각화/frontend` ← **반드시 변경** ("Edit" 버튼 클릭) |
| Build Command | `npm run build` (자동) |
| Output Directory | `dist` (자동) |
| Install Command | `npm install` (자동) |

### ⚠️ Branch 설정 (배포 후 반드시 변경)
- 첫 배포 후 자동으로 `main` 브랜치를 production으로 잡음
- 우리 작업물은 `contest-v2-14th` 브랜치에 있음
- **Settings → Git → Production Branch**를 `contest-v2-14th`로 변경
- 그 다음 다시 Deploy 트리거

## 3️⃣ Deploy 클릭

- 1~2분 대기 → `https://datacenter-contest-v2-xxx.vercel.app` 형태 URL 발급
- 이후 `contest-v2-14th` 브랜치에 push할 때마다 **자동 재배포**

---

## 배포 후 확인

대시보드가 정상 동작하면:
- [ ] 4개 탭(지도/대시보드/비교/정책) 모두 진입 가능
- [ ] 지도에 109개 마커 표시
- [ ] 가중치 슬라이더 조절 시 즉시 재계산
- [ ] 시나리오 프리셋 3개 동작
- [ ] 정책 탭에서 5개 제언 + 출처표 노출

문제 있으면 Vercel 빌드 로그(Deployments → 해당 배포 → Logs) 확인.

## 나중에 Node 업그레이드 후 CLI 쓰려면

```powershell
# nvm-windows 설치: https://github.com/coreybutler/nvm-windows/releases
nvm install 20
nvm use 20
node --version   # v20.x 확인

# 그 다음
cd C:\dev\datacenter_contest\04_시각화\frontend
vercel login         # 브라우저로 GitHub 인증
vercel link          # 기존 Vercel 프로젝트와 연결
vercel --prod        # production 배포
```

## 대안: 만약 Vercel이 막히면

### Netlify (거의 동일)
- https://app.netlify.com/start → GitHub 연동
- **Base directory**: `04_시각화/frontend`
- **Build command**: `npm run build`
- **Publish directory**: `04_시각화/frontend/dist`
- **Branch**: `contest-v2-14th`

### GitHub Pages
- `vite.config.ts`에 `base: '/datacenter_select/'` 추가 필요 (레포 이름)
- Actions로 빌드 후 `gh-pages` 브랜치에 publish
- Vercel/Netlify가 훨씬 편함
