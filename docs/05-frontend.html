왜 Vue (vs Streamlit)?

이전 세션에서 Streamlit 후보가 있었으나 다음 비교로 Vue 확정:

기준Vue 3 (선택)Streamlit (대안)
반응성슬라이더↔지도 ms 단위매 슬라이더에 페이지 재실행
지도 인터랙션MapLibre 풀 컨트롤folium iframe 제약
디자인 자유도Tailwind + 커스텀위젯 디자인 제약
배포정적 호스팅 (영구 무료)Streamlit Cloud 리소스 제한
평가 인상제품·서비스 도구분석 노트북 느낌

MapView.vue — MapLibre

  • OSM raster 타일 (저작권 표시)
  • 시군별 마커: 점수에 따라 색상(HSL hue 0→140) + 크기(8→30px)
  • 호버: scale 1.25 / 클릭: store.toggleCompare(name)
  • 비교 모드 선택 시 파란 테두리 강조
  • 좌하단 점수 색상 범례

WeightPanel.vue — 슬라이더 + Top 10

  • 9개 factor 슬라이더 (range 0~50)
  • % 표시는 weights[key] / weightSum × 100 (실시간 상대치)
  • Top 10 리스트 — 클릭하면 비교 모드 추가/제거
  • "기본값으로 초기화" 버튼

Dashboard.vue — 3개 차트

  1. Top 20 막대: 점수별 색상
  2. 실거래가 vs 종합점수 산점: 토지비-점수 상관 시각화
  3. Top 10 vs Bottom 10 요인 평균: 강점/약점 패턴

CompareView.vue — 비교 모드

  • 최대 3개 후보지 동시 비교
  • 레이더 차트: 9개 지표 정규화 점수 (대전 vs 제천 같은 패턴 차이 즉시 보임)
  • 표: 종합점수, 순위, 9개 raw 값 (단위 포함)
  • "비교 초기화" 버튼

HMR 주의사항

가중치 dynamic 로드: 코드 수정으로 store 구조 바뀌면 Pinia state가 stale. 브라우저 강제 리로드(F5) 필요. scoring.ts에서 weights는 load() 시점에 payload.factors[].default_weight로 초기화됨.
ECharts 0×0 버그: v-show로 숨긴 컨테이너에서 mount되면 canvas 크기가 0이 됨. Dashboard/Compare는 v-if로 매번 새로 mount하여 회피. MapView만 KeepAlive (지도 줌 상태 보존).
