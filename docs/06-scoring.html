06점수 계산 로직

raw 값 → 정규화 → 방향 반전 → 가중평균. 슬라이더 움직이면 즉시 재계산.

전체 흐름

raw[N×9 행렬]                  ┐
  └─ min-max 정규화 ──→ norm[N×9 ∈ [0,1]]
       └─ direction이 lower_is_better면 (1 − v)로 반전
            └─ weights × norm 가중평균 ──→ score[N ∈ [0,1]]
                 └─ 내림차순 정렬 ──→ rank

(N = 109 시군)

1단계: min-max 정규화

각 지표별로 109개 시군 분포에서 min/max 사용.

function normalizeMinMax(values: number[]): number[] {
  const min = Math.min(...values);
  const max = Math.max(...values);
  const span = max - min || 1;          // div-by-zero 방지
  return values.map((v) => (v - min) / span);
}
왜 z-score 아니고 min-max? 사용자에게 [0,1] 절대 척도가 직관적. 레이더 차트에서 "최고치 1, 최저치 0" 한눈에 보임. 정규분포 가정 안 해도 됨 (회사수처럼 한쪽 꼬리 긴 분포에 강함).

2단계: 방향 반전

지표마다 "높을수록 좋음" / "낮을수록 좋음"이 다름. 모든 점수를 "높을수록 좋음"으로 통일.

for (const f of factors.value) {
  const raw = regions.value.map((r) => r.factors[f.key]);
  const n = normalizeMinMax(raw);
  out[f.key] = f.direction === "lower_is_better" ? n.map((v) => 1 - v) : n;
}

방향별 지표

방향지표의미
↑ higher전력 안정성, 산업 집적도, 통신 인프라, IT 인력, 지역활력, 재생에너지그대로 사용
↓ lower자연재해 위험, 평균기온, 토지 비용1 − v로 반전

3단계: 가중평균 (Pinia computed)

const scored = computed<ScoredRegion[]>(() => {
  const sum = weightSum.value || 1;
  return regions.value.map((r, i) => {
    const componentScores = {};
    let total = 0;
    for (const f of factors.value) {
      const v = normalized.value[f.key][i];     // [0,1]
      const w = weights.value[f.key] / sum;      // 상대 가중치
      componentScores[f.key] = v;
      total += v * w;
    }
    return { ...r, score: total, rank: 0, componentScores };
  });
});

weightSum으로 정규화하기 때문에 사용자가 슬라이더로 합을 100% 안 맞춰도 됨.

4단계: 순위 부여

const sorted = [...scored.value].sort((a, b) => b.score - a.score);
sorted.forEach((r, i) => (r.rank = i + 1));   // 1, 2, 3, ...
return scored.value;   // 원본 순서 유지하되 rank 채움

반응성 보장

Pinia computed는 dependency 추적이 자동. 슬라이더가 weights[key]를 갱신하면:

  1. weightSum 재계산 (의존: weights)
  2. scored 재계산 (의존: weights, weightSum, normalized)
  3. topN 재계산 (의존: scored)
  4. compared 재계산 (의존: scored, compareSet)
  5. MapView 마커 색·크기 재렌더, Dashboard 차트 재드로우, CompareView 레이더 재드로우

모두 한 frame 안에서 일어남 — 슬라이더 ↔ 지도 ms 단위 연동.
