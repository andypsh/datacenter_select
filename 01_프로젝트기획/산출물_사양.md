# 산출물 사양 (제품·서비스 개발 부문)

## 메인 산출물: AI 데이터센터 입지 의사결정 지원 대시보드

> 정책 담당자/투자자가 **직접 가중치를 조정하며 후보지를 탐색**할 수 있는 인터랙티브 웹앱.
> 단순 분석 보고서를 뛰어넘는 "도구"로 포지셔닝 → 제품·서비스 개발 부문 정합성 확보.

---

## 기술 스택 (확정)

| 영역 | 선택 | 이유 |
|---|---|---|
| 프론트엔드 | **Vue 3 + Vite + TypeScript** | 컴포넌트화·반응성 강점. Pinia로 가중치 상태 관리. 빌드 결과는 정적 자산이라 어디든 배포 가능 |
| 지도 | **MapLibre GL** | OSM 호환 오픈소스, 모던 WebGL 렌더링. 시군구 폴리곤·히트맵·마커 자유 |
| 차트 | **ECharts (vue-echarts)** | 막대/산점/레이더/시계열 등 풍부. 다크 테마와 한글 폰트 호환 우수 |
| 상태 | **Pinia** | 가중치 슬라이더 ↔ 점수 계산 ↔ 지도/차트 갱신을 자연스럽게 |
| 스타일 | **Tailwind CSS** | 빠른 UI 반복, 다크 모드 일관성 |
| 분석 | pandas, geopandas, numpy | 시군구 단위 공간조인 + 정규화 + 가중평균 |
| 배포 | **GitHub Pages** (또는 Vercel/Cloudflare Pages) | 정적 자산만 호스팅, 무료, 커밋 시 자동 배포 |

### 왜 Vue인가 (이전 검토: Streamlit)

| 기준 | Vue 3 (선택) | Streamlit (대안) |
|---|---|---|
| 개발 속도 | 컴포넌트 재사용 후 빠름 | 초기 진입 가장 빠름 |
| 반응성 | 슬라이더↔지도 ms 단위 갱신 | 매 슬라이더에 페이지 재실행 |
| 지도 인터랙션 | MapLibre 풀 컨트롤 | folium iframe 제약 |
| 디자인 자유도 | Tailwind + 커스텀 컴포넌트 | 위젯 디자인 제약 |
| 배포 | 정적 호스팅 (영구 무료) | Streamlit Cloud (할당 자원 제한) |
| 평가 인상 | 자체 제작 인터랙티브 도구 | 데이터 분석 노트북 느낌 |

→ **"제품·서비스 개발 부문"** 정합성에서 Vue가 더 강함.
